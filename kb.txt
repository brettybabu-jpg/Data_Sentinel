data integrity
Data integrity means keeping data accurate and consistent over time.

why data integrity
It prevents incorrect or duplicate data and ensures reliability.

database
A database is an organized collection of data.

dbms
A DBMS is software used to store and manage data.

table
A table stores data in rows and columns.

row
A row represents a single record.

column
A column represents an attribute.

record
A record is another name for a row.

field
A field is a single data value in a record.

sql
SQL is used to interact with databases.

query
A query is a request for data.

schema
Schema defines the structure of a database.

instance
Instance is the actual data stored in the database.

attribute
Attribute is another name for column.

tuple
A tuple is a row in a table.

domain
Domain defines allowed values for a column.

null
NULL means no value.

primary key
Primary key uniquely identifies each record.

foreign key
Foreign key links two tables.

candidate key
A candidate key can become a primary key.

super key
A super key uniquely identifies records.

composite key
A key made of multiple columns.

alternate key
A candidate key not chosen as primary.

surrogate key
An artificial key generated by system.

natural key
A real-world meaningful key.

constraint
Constraints are rules applied to data.

not null
Ensures column cannot have empty value.

unique
Ensures all values are different.

check
Ensures a condition is satisfied.

default
Assigns default value if none provided.

referential integrity
Maintains valid relationships between tables.

entity integrity
Ensures primary key is not null.

domain integrity
Ensures valid data type and range.

user defined integrity
Custom rules defined by users.

data consistency
Data remains same across database.

data redundancy
Duplicate data storage.

data accuracy
Data is correct and error-free.

data reliability
Data can be trusted.

data security
Protects data from unauthorized access.

data validation
Checks correctness of data.

data corruption
Data becomes damaged or incorrect.

data governance
Manages data quality and security.

data lifecycle
Stages of data from creation to deletion.

audit trail
Tracks data changes.

normalization
Organizes data to reduce redundancy.

1nf
Removes repeating groups.

2nf
Removes partial dependency.

3nf
Removes transitive dependency.

bcNF
A stronger version of 3NF.

denormalization
Adds redundancy for performance.

transaction
A group of operations.

acid
Atomicity Consistency Isolation Durability.

atomicity
All operations succeed or none.

consistency
Database remains valid.

isolation
Transactions do not interfere.

durability
Changes are permanent.

commit
Saves changes.

rollback
Undoes changes.

savepoint
Temporary point in transaction.

index
Improves search speed.

clustered index
Sorts data physically.

non clustered index
Separate structure for indexing.

view
Virtual table based on query.

materialized view
Stores query result physically.

join
Combines data from multiple tables.

inner join
Returns matching records.

left join
Returns all left table records.

right join
Returns all right table records.

full join
Returns all records from both tables.

self join
Table joined with itself.

cross join
Cartesian product of tables.

subquery
Query inside another query.

group by
Groups rows with same values.

having
Filters grouped data.

order by
Sorts results.

distinct
Removes duplicate values.

limit
Restricts number of results.

union
Combines results of queries.

intersect
Returns common results.

except
Returns non-common results.

trigger
Automatically executes on event.

procedure
Stored block of SQL code.

function
Returns value after execution.

cursor
Processes rows one by one.

lock
Prevents data conflict.

deadlock
Two transactions waiting for each other.

backup
Copy of database.

recovery
Restoring database after failure.

sharding
Splitting database into parts.

replication
Copying data across servers.
